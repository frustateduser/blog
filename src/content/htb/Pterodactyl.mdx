### Exploitation

To obtain the initial foothold, I found that the pterodactyl **terodactyl Panel v1.11.10** is running which is vulnerable to RCE via [CVE-2025-49132](https://nvd.nist.gov/vuln/detail/CVE-2025-49132).

I got the exploit from searching the web and found the following exploit code on [GitHub](https://github.com/malw0re/CVE-2025-49132---Pterodactyl-RCE-HTB-Season-10-/blob/main/ape1.py):

    print(f"{Colors.BLUE}[*]{Colors.END} {msg}")

def clean_output(raw_output):
    """
    Clean PEAR config output to extract actual command results

    if not raw_output:
        return None
    
    lines = raw_output.split('\n')
    results = []
    
    for line in lines:

            clean = re.sub(r'\x1b\[[0-9;]*m', '', line)  # Remove ANSI codes
            clean = re.sub(r'.*?(uid=\d+.*?)\s+uid=.*', r'\1', clean)  # Remove duplicates
            clean = clean.strip()
            
            if clean and clean not in results:
                results.append(clean)
    
    # If we found specific patterns, return them
    if results:
        return '\n'.join(results)
    
    # Otherwise try regex extraction
    patterns = [
        r'(uid=\d+\([^)]+\)[^\n<]+)',  # uid output
        r'(HTB\{[^}]+\})',  # HTB flags
        r'(root:[^:]+:[^:]+:[^:]+:[^:]+:[^:]+:[^\n]+)',  # /etc/passwd format
        r'(total\s+\d+.*)',  # ls output

    for pattern in patterns:
        match = re.search(pattern, raw_output, re.MULTILINE)

    if not raw_output.strip().startswith('{'):

This email indicates that there is some unusual activity with `udisksd` which is a daemon used for managing disks. This could be a potential vector for privilege escalation.
`udisksctl` is a command-line tool that interacts with the `udisksd` daemon. If there are vulnerabilities in `udisksd`, it might be possible to exploit them to gain elevated privileges.
But it was not the case here, as the service was not accessible to do so I explioited the [CVE-2025-6018](https://nvd.nist.gov/vuln/detail/CVE-2025-6018).  This flaw allows an unprivileged 
local attacker (for example, a user logged in via SSH) to obtain the elevated privileges normally reserved for a physically present, "allow_active" user.
